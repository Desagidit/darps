# Authoring best practices

## Protect truth structurally

- Put hidden knowledge behind `when` conditions so it is absent from unrelated
  contexts.
- Connect every gameplay discovery to a fact and a valid source.
- Treat `why` and `tell` as performance direction, not security.
- Keep spoiler-bearing ground truth in `vars.yaml` and gated knowledge.

## Write resilient characters

- Give every character enough background to answer ordinary personal
  questions consistently.
- Specify voice through usable behavior, not adjectives alone.
- Give concealed information a visible tell.
- Write both sides of flag transitions so a character does not become silent
  when a lie expires.

## Make attitudes rewarding

- State what raises, lowers, and leaves each track unchanged.
- Use slower speeds for relationships intended to be earned.
- Add character guidance only for exceptions to the shared track rules.
- Ensure `track_prose` covers the full reachable range.

## Preserve solvability

- Give critical facts more than one plausible route where appropriate.
- Do not rely exclusively on a relationship gate if alienating one character
  would permanently block the host story.
- Validate after every edit and take unreachable-fact warnings seriously.
- Test the exact player vocabulary used by search triggers and aliases.

## Keep the boundary clean

Packs describe narrative content. Inventory effects, movement, combat,
cutscenes, quest completion, and flag mutation belong to the host game.
